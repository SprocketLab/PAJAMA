Codifying the Judge: Scalable Evaluation via Program Distillation

Tzu-Heng Huang, Shengqi Qiu, Frederic Sala
University of Wisconsin-Madison
ICML 2025 Workshop on Programmatic Representations for Agent Learning
Paper Code arXiv 🤗 Dataset

PAJAMA replaces expensive LLM-as-a-judge calls with lightweight Python programs distilled from evaluation criteria, then aggregates their predictions via weak supervision to produce scalable, transparent, and cost-effective verdicts.

Abstract

LLM-as-a-judge is everywhere but can suffer from high cost, significant latency, and opaque decisions—limitations that undermine its scalability and reliability. We address these with a simple, efficient alternative: program distillation.

We propose PAJAMA, a framework for scalable, cost-effective evaluation. Instead of calling an LLM judge for every sample, PAJAMA distills evaluation criteria into lightweight Python programs and aggregates their predictions using weak supervision to produce final verdicts. PAJAMA generates 80 judge programs (10 evaluation rubrics × 8 variants each). Each program takes a query and a pair of responses, then returns a score using only built-in Python—no ML libraries—making them fast, deterministic, and fully interpretable.

Program outputs are aggregated using a weak supervision label model that learns per-program accuracy weights, filters low-quality programs, and produces calibrated preference predictions. The resulting labels can optionally be used to fine-tune a lightweight reward model. Across benchmarks, PAJAMA achieves strong agreement with human judgments at a fraction of the cost of LLM-based judges.

PAJAMA achieves competitive accuracy with human judges at significantly lower cost across multiple evaluation benchmarks.

80 judge programs are generated from 10 evaluation rubrics, each with 8 structural variants for diversity.

A weak supervision label model learns per-program accuracy weights and produces calibrated final predictions.

Aggregated labels from PAJAMA can distill a lightweight reward model (Qwen2.5-3B) with strong downstream performance.

How PAJAMA Works

  1. Program Generation — Generates 80 judge programs (10 evaluation rubrics × 8 variants each). Each program takes a query and a response, then returns a float score using only built-in Python (no ML libraries), making them fast and deterministic.
  2. Program Execution — Each program scores both responses in a pairwise comparison, producing a label matrix over the dataset.
  3. Program Output Modeling — Per-program thresholds are tuned on a validation set, low-accuracy programs are filtered, and weak supervision's Label Model learns per-program accuracy weights to produce final preference predictions.
  4. Reward Model Training (Optional) — The predicted labels are used to fine-tune a Qwen2.5-3B reward model for downstream use.

Interactive Demo

PAJAMA ships with an interactive Evaluation Studio built in Streamlit. Inspect individual judge programs, edit them, regenerate via chat, and download labeled datasets.
